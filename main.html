Git Topic

Git adalah salah satu sistem pengontrol versi (Version Control System) pada proyek perangkat lunak yang diciptakan oleh Linus Torvalds. Pengontrol versi bertugas mencatat setiap perubahan pada file proyek yang dikerjakan oleh banyak orang maupun sendiri. Git dikenal juga dengan distributed revision control (VCS terdistribusi), artinya penyimpanan database Git tidak hanya berada dalam satu tempat saja. Semua orang yang terlibat dalam pengkodean proyek akan menyimpan database Git, sehingga akan memudahkan dalam mengelola proyek baik online maupun offline. Git memantau semua perubahan yang terjadi pada file proyek dan menyimpannya ke dalam database. Git hanya akan menyimpan delta perubahannya saja, dia tidak akan menyimpan seluruh isi file yang akan memakan banyak memori. Git memungkinkan kita kembali ke versi revisi yang kita inginkan.

Git memiliki beberapa kelebihan, antara lain:
- Bisa menyimpan seluruh versi source code;
- Bisa paham cara kolaborasi dalam proyek; - Bisa ikut berkontribusi ke poryek open-source;
- Lebih aman digunakan untuk kolaborasi, karena kita bisa tahu apa yang diubah dan siapa yang mengubahnya.

Git memiliki beberapa alat atau tools yang dapat digunakan untuk melakukan operasi-operasi pada proyek, antara lain:
- Git init : untuk membuat repository lokal baru
- Git add : untuk menambahkan file ke staging area
- Git commit : untuk menyimpan perubahan ke repository lokal
- Git push : untuk mengirimkan perubahan ke repository remote
- Git pull : untuk mengambil perubahan dari repository remote
- Git fetch : untuk mengambil informasi dari repository remote tanpa menggabungkannya
- Git status : untuk melihat status dari file di working directory dan staging area
- Git log : untuk melihat riwayat commit
- Git diff : untuk melihat perbedaan antara file di working directory dan staging area
- Git branch : untuk melihat seluruh branch yang ada pada repository
- Git checkout : untuk menukar branch yang aktif dengan branch yang dipilih
- Git merge : untuk menggabungkan branch yang aktif dan branch yang dipilih
- Git clone : untuk membuat salinan repository lokal
